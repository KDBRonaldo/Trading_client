# 项目分工

本项目当前已经完成前端基础框架、页面展示、本地 mock 数据和外部接口预留。接下来重点是和其他模块进行接口联调、数据传递和最终部署演示。

## 分工一：资金账户/证券账户系统对接负责人

### 负责内容

负责和资金账户、证券账户系统对接，保证账户相关数据能够真实交互。

主要任务：

- 登录时调用资金账户系统校验资金账户卡号和交易密码。
- 查询资金账户可用资金、冻结资金、账户状态。
- 查询证券账户持仓、可卖数量、持仓成本。
- 修改交易密码和取款密码。
- 买入委托时冻结资金。
- 买入失败或撤单时释放资金。
- 卖出委托时冻结股票。
- 卖出失败或撤单时释放股票。

```

```

### 交付标准

- 能使用真实资金账户系统登录。
- 页面能显示真实资金余额和冻结资金。
- 页面能显示真实证券持仓。
- 买入/卖出时冻结资源成功。
- 撤单或失败时释放资源成功。
- 接口失败时页面能给出明确提示。

## 分工二：中央交易系统/交易管理系统对接负责人

### 负责内容

负责和交易管理系统、中央交易系统对接，保证行情、委托、撤单和成交结果能够真实交互。

主要任务：

- 买入/卖出委托提交前调用交易管理系统审查。
- 审查失败时阻止提交，并展示失败原因。
- 从中央交易系统查询股票行情。
- 向中央交易系统提交买入/卖出委托。
- 向中央交易系统发起撤单请求。
- 从中央交易系统同步成交结果。
- 将中央交易系统返回的订单状态转换成前端状态。

### 主要修改文件

```text
js/management-api.js
js/central-api.js
js/config.js
INTERFACES.md
```

必要时修改：

```text
js/business.js
```

### 重点函数

```js
reviewOrderByManagement()
fetchQuotes()
submitOrderToCentral()
cancelOrderInCentral()
fetchOrderResultFromCentral()
normalizeStockQuote()
normalizeOrderStatus()
```

### 交付标准

- 行情查询来自中央交易系统。
- 买入/卖出委托先经过交易管理系统审查。
- 审查通过后能提交中央交易系统。
- 撤单请求由中央交易系统返回结果。
- 成交结果由中央交易系统返回，前端不能自行决定成交。
- 状态字段能正确显示为“未成交、部分成交、已成交、已撤销、已拒绝”等。

## 分工三：机动支持人员（2人）

### 负责内容

机动支持人员暂不固定到某一个外部系统接口，主要根据项目推进过程中出现的问题随时接任务。

可能负责的任务包括：

- 业务流程整合：把账户接口、交易管理接口、中央交易接口串成完整流程。
- 联调问题排查：定位问题来自前端参数、接口返回、跨域、网络、数据库还是外部系统状态。
- 部署支持：整理一台服务器上各模块的启动方式、端口、访问路径和反向代理方案。
- 数据库支持：维护交易客户端本地数据库表、建表脚本和持久化方案。
- 测试准备：虽然正式测试任务后面会单独做，但可以提前整理联调测试清单和演示路径。
- 数据一致性检查：检查资金、持仓、委托、成交在页面上的显示是否同步。
- 数据库一致性检查：检查 `login_session`、`order_record`、`trade_record`、`price_alert`、`alert_notification` 等表是否和页面、外部系统状态一致。
- mock/真实接口切换支持：确认本地演示模式和真实接口模式都能使用。
