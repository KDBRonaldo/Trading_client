# 股票交易系统 - 交易客户端原型

这是根据需求报告和设计报告搭建的交易客户端前端原型。当前版本是纯前端实现，不依赖后端服务，使用 `localStorage` 模拟资金账户、证券持仓、行情、委托、成交和价格提醒数据。

## 运行方式

直接在浏览器中打开 `index.html`。

演示账户：

- 资金账户卡号：`6222026000000001`
- 交易密码：`123456`
- 取款密码：`654321`

## 已覆盖功能

- 登录校验：16位资金账户卡号、6位交易密码、首次登录认证提示、连续错误锁定。
- 账户查询：可用资金、冻结资金、总资产估算、证券持仓和持仓盈亏。
- 行情查询：按6位代码或股票名称模糊查询，展示最新价、买卖盘口、最高最低价、涨跌停价和公告。
- 买入委托：校验股票状态、价格涨跌停范围、数量100股整数倍、可用资金，成功后冻结资金。
- 卖出委托：校验持仓、价格涨跌停范围、数量100股整数倍、可卖数量，成功后冻结股票。
- 委托撤销：未成交委托可撤销，撤销后释放冻结资金或股票。
- 成交回报：可手动模拟成交，成交后刷新资金、持仓、委托状态和成交列表。
- 价格提醒：新增、查看、删除提醒规则，刷新行情时自动检查触发条件。
- 密码修改：支持交易密码和取款密码，校验原密码、新密码格式、重复密码和二次确认。
- 会话控制：模拟30分钟无操作超时，超时后回到登录页。

## 后续对接建议

如果后续要接入 Java/Spring Boot 后端，可以优先把 `app.js` 中的模拟数据操作替换成接口调用，保留现有的表单校验和页面渲染结构。建议接口按设计报告中的外部系统边界拆分为资金账户、证券账户、中央交易、行情发布和价格提醒五类服务。
